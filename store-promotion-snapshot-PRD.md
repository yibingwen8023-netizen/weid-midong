## 一、需求背景

### 1.1 业务场景

「门店项目推广配置」页面记录了各门店各项目的推广开关状态。运营人员会频繁调整这些开关配置，为了：

1. **追溯变更历史** — 查看某天所有门店的推广配置是什么样的
2. **问题排查** — 发现推广异常时，可回溯到特定日期的快照数据
3. **数据对比** — 对比不同日期的配置差异，分析推广策略调整效果

系统需要每天自动生成一份配置快照，并提供一个独立的查询页面供运营人员查看历史快照。

### 1.2 与现有页面的关系

| 页面 | 用途 | 关系 |
|------|------|------|
| 门店项目推广配置（已有） | 管理当前实时推广开关配置 | **数据源**：其当前状态每天被快照记录 |
| **推广配置快照查询（新增）** | 查看历史的每日推广配置快照 | **查询入口**：只读展示，不可修改 |

## 四、交互细节

### 4.1 日期选择与查询

1. 页面加载时，默认选中「最近7天」，自动加载最新的快照详情
2. 切换快捷日期按钮 → 更新日期范围 → 重新加载快照
3. 手动修改日期范围 → 点击「搜索」→ 更新快照
4. 点击「重置」→ 恢复最近7天默认值

### 4.2 快照切换

1. 日期范围选定后，展示该范围内的**最新一天**的快照详情
2. 如果该范围内有多天快照，可在表格上方通过日期导航切换：
   - 显示 `◀ 2026-04-05 | 2026-04-06（当前） ▶`
   - 左右箭头切换前/后一天的快照
3. 切换日期后，表格数据和统计卡片同步更新

### 4.3 筛选交互

1. 城市和门店名称筛选作用于快照表格中的门店行
2. 推广状态下拉筛选门店行
3. 筛选只影响表格显示，不影响统计卡片数据

### 4.4 只读提示

1. 表格区域标题处显示「📋 快照数据（只读）」提示
2. 开关列用 ✓/✗ 图标替代可操作的Switch组件

## 五、与门店项目推广配置页面的差异对比

| 维度 | 门店项目推广配置 | 推广配置快照查询 |
|------|-----------------|-----------------|
| 数据性质 | 实时可编辑 | 历史只读 |
| 操作方式 | Switch开关可切换 | ✓/✗ 图标展示 |
| 批量操作 | 有（批量开启/关闭） | 无 |
| 全选Checkbox | 有 | 无 |
| 日期选择 | 无 | 有（核心筛选条件） |
| 统计卡片 | 无 | 有（3个概览卡） |
| 快照导航 | 无 | 有（◀ 日期 ▶） |
| 标签页 | 共享标签组 | 独立页面或新标签页 |

生成最近7天（2026-03-31 ~ 2026-04-06）的快照数据，每天一份。每天的数据在前一天基础上随机变化2-3个门店的项目开关，模拟真实的配置变更过程。

### 6.4 JavaScript 功能

1. **快捷日期切换**：点击快捷按钮更新日期范围
2. **快照数据加载**：根据日期生成/加载对应的模拟快照
3. **快照日期导航**：◀ ▶ 切换查看不同日期的快照
4. **统计卡片更新**：根据当前快照计算总门店数、开启数、关闭数
5. **筛选功能**：城市、门店名称、推广状态筛选
6. **表格渲染**：只读模式渲染 ✓/✗ 状态
