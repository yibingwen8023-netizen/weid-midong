## 数据逻辑说明

### 数据结构特点
1. **床位管理核心指标**：
   - 释放床位：当月释放的床位数量
   - 剩余床位：当月剩余可用床位
   - 已用床位：实际使用的床位数量
   - 爽约相关：爽约数和爽约率（红色负值显示）

2. **层级关系**：
   - 每个员工都有固定的"30床位"总量
   - 释放床位 + 剩余床位 = 30（假设）
   - 已用床位 ≤ 释放床位

### 业务逻辑
1. **床位分配**：每个前端员工每月有30个床位额度
2. **床位释放**：员工可以释放部分床位（0-30之间）
3. **床位使用**：实际占用的床位数量
4. **爽约跟踪**：排客后未到店的情况统计
5. **到店转化**：实际到店客户数量

### 数据显示规则
1. **默认数据**：0到30之间的床位数值
2. **负值显示**：爽约数和爽约率用红色显示
3. **平均/合计行**：汇总统计数据
4. **操作列**："设置"按钮用于调整床位参数

## 页面功能

### 主要操作
1. **床位设置**：调整员工的释放床位数量
2. **一键设置**：批量设置整个大区的床位参数
3. **数据筛选**：按大区、部门、小组、人员多维度筛选
4. **层级切换**：在组员/小组/大区三个层级间切换视图

### 交互特性
1. **响应式设计**：适应不同屏幕尺寸
2. **状态指示**：活动状态的视觉反馈
3. **数据说明**：帮助图标提供字段说明
4. **批量操作**：支持全选和批量设置

## 设计风格

### UI特点
1. **暗色主题**：左侧菜单和顶部导航采用深色背景
2. **现代简约**：图标采用Remix Icon，清晰易识别
3. **数据表格**：专业的数据表格设计，支持多级表头
4. **色彩系统**：
   - 主色调：蓝色（#1890ff）
   - 强调色：红色（#f5222d）用于负值
   - 中性色：灰色调用于背景和边框

### 布局特点
1. **侧边导航**：固定宽度，支持滚动
2. **内容区域**：自适应宽度，支持滚动
3. **筛选区域**：卡片式设计，区分不同筛选条件
4. **表格区域**：清晰的视觉层级，易于数据对比
