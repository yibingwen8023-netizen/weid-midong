# 门店项目推广配置 - 需求分析文档

**文档版本**: v1.0  
**创建日期**: 2026-04-06  
**需求类型**: 新增页面  
**所属模块**: 产品 → 门店项目推广配置

---

## 一、需求背景

### 1.1 业务场景

微动在全国拥有多家线下门店，每个门店提供多种美容/医美服务项目（如 S10、M10、半永久女、半永久男、K10、K20、K30、S10N、S10N1、S10Y、S20、S30 等）。部分项目需要通过线上渠道（如推广链接、广告投放）进行推广引流，因此需要一个后台配置页面，让运营人员能灵活控制每个门店的每个项目是否开启推广功能。

### 1.2 与现有页面的关系

| 页面 | 用途 | 关系 |
|------|------|------|
| 门店预约项目（已有） | 控制「门店是否开启预约」及「各项目是否可预约」 | **前置配置**：先决定哪些项目可用 |
| **门店项目推广配置（新增）** | 控制「各门店的各项目是否开启推广」 | **下游配置**：在可用项目基础上决定哪些做推广 |

两个页面结构高度相似（都是 门店×项目 的矩阵表格），区别在于：
- 门店预约项目：管理「预约功能开关」
- 门店项目推广配置：管理「推广功能开关」

## 二、功能需求

### 2.1 页面导航定位

```
产品（一级菜单，展开状态）
├── 预约意向上架
├── 门店预约项目
└── 门店项目推广配置  ← 新增
```

面包屑路径：`首页 / 产品 / 门店项目推广配置`

标签页：与「门店预约项目」共享同一组标签页，新增一个标签：

```
┌─────────────────┬──────────────────────┐
│ 门店预约项目      │ 门店项目推广配置       │
└─────────────────┴──────────────────────┘
```

### 3.3 推广状态计算规则

```
如果该门店有 ≥1 个项目开关为「开」 → 推广状态 = 开启（绿色）
如果该门店所有项目开关均为「关」 → 推广状态 = 关闭（红色）
```

## 四、交互细节

### 4.1 开关切换交互

1. 用户点击某个项目的 Switch 开关
2. 开关状态立即切换（带 0.3s 滑动动画）
3. 自动重新计算该门店的推广状态标签
4. 模拟数据场景下，状态即时更新

### 4.2 批量操作交互

1. 用户勾选一个或多个门店行
2. 点击「批量开启推广」按钮
3. 所有选中门店的所有项目开关切换为「开」
4. 推广状态全部变为「开启」
5. 反之同理

### 4.3 筛选交互

1. 点击城市/门店名称标签 → 单选切换
2. 点击「展开」→ 显示更多选项
3. 选择下拉筛选条件
4. 点击「搜索」→ 表格更新
5. 点击「重置」→ 所有筛选条件恢复默认

### 4.4 标签页切换

1. 点击「门店预约项目」标签 → 切换到门店预约项目页面
2. 点击「门店项目推广配置」标签 → 当前页面

## 五、与门店预约项目页面的差异对比

| 维度 | 门店预约项目 | 门店项目推广配置 |
|------|-------------|-----------------|
| 管理对象 | 门店预约开关 + 各项目预约开关 | 各项目推广开关 |
| 操作按钮 | 快速修改预约、导出、批量导入 | 批量开启推广、导出 |
| 状态标签含义 | 门店预约是否开启 | 门店是否有项目在推广 |
| 筛选条件中的下拉 | 门店预约状态、项目 | 推广状态、项目 |
| 面包屑 | 首页/产品/门店预约项目 | 首页/产品/门店项目推广配置 |
| 表格列 | 推广状态列 + 12个项目列 | 推广状态列 + 12个项目列 |
| 项目列数 | 12 | 12（相同） |
